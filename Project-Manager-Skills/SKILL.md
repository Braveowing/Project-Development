# PMP 项目管理技能

基于 PMBOK 第七版原则和第六版过程组框架，覆盖项目全生命周期管理。

## 核心原则

- **价值导向**：始终以交付业务价值为最终目标，而非仅完成交付物
- **渐进明细**：项目信息随推进逐步清晰，规划应迭代更新
- **干系人参与**：持续识别和管理相关方期望
- **适应型方法**：根据项目特征选择预测型、混合型或适应型方法

## 项目全生命周期

当用户启动新项目或提及项目事务时，按以下阶段推进：

### 一、启动阶段

> 📖 详细指导见 `references/initiating-guide.md`，包含：输入/工具/输出、权力利益方格、专家判断方法等。

1. **编制项目章程** — 明确项目目标、高层级需求、里程碑、预算上限、项目经理权限
   - 使用 `references/project-charter.md` 模板
   - 存档到 `memory/projects/<project-name>/charter.md`
2. **识别相关方** — 梳理所有受项目影响的个人和组织，记录他们的期望、影响力和参与程度
3. **确认项目可行性** — 评估商业论证是否成立

### 二、规划阶段

> 📖 详细指导见 `references/planning-guide.md`，包含：WBS分解原则、CPM关键路径法、EVM挣值管理、七种质量工具、RACI矩阵、风险概率影响矩阵、合同类型选择等。

按十大知识领域进行规划，关键产出物见 `references/planning-checklist.md`：

| 知识领域 | 核心产出 |
|----------|---------|
| 范围 | WBS、范围基准 |
| 进度 | 甘特图、里程碑计划、进度基准 |
| 成本 | 预算估算、成本基准 |
| 质量 | 质量标准、验收标准 |
| 资源 | 资源日历、团队职责分配 |
| 沟通 | 沟通计划、汇报机制 |
| 风险 | 风险登记册、应对策略 |
| 采购 | 采购计划（如需要） |
| 相关方 | 相关方参与计划 |
| 整合 | 项目管理计划（汇总以上所有） |

规划产出物存档到 `memory/projects/<project-name>/plan/`。

### 三、执行与监控阶段

> 📖 详细指导见 `references/monitoring-guide.md`，包含：EVM完工预测（EAC/ETC/VAC/TCPI）、塔克曼团队发展理论、变更控制流程、问题升级路径、预警信号体系等。

日常项目管理围绕以下活动展开：

1. **进度跟踪** — 对比实际进展与基准，识别偏差
   - 定期（周/双周）生成项目状态报告
   - 使用 `references/status-report.md` 模板
2. **风险监控** — 审查风险登记册，更新风险状态，识别新风险
3. **变更管理** — 所有变更走变更控制流程：提出→评估→审批→实施→验证
4. **问题管理** — 区分风险（未来可能发生）和问题（已发生），问题需立即处理
5. **质量保证** — 确保交付物符合验收标准
6. **相关方沟通** — 按沟通计划定期同步项目状态

**关键判断规则：**
- 偏差 ≤ 10%：记录并在下次汇报中说明
- 偏差 > 10%：立即预警，建议纠正措施
- 偏差 > 25%：视为重大风险，启动变更控制流程

### 四、收尾阶段

> 📖 详细指导见 `references/closing-guide.md`，包含：验收流程、经验教训总结会议议程、组织过程资产更新、收尾常见陷阱等。

1. **验收交付物** — 逐项对照范围基准和验收标准
2. **移交成果** — 确保运营团队具备接收条件
3. **总结经验教训** — 记录哪些做法有效、哪些无效
4. **归档项目文件** — 整理并存档所有项目文档
5. **释放资源** — 正式释放团队成员和物料资源

使用 `references/closing-checklist.md` 模板。

## 项目文件结构

每个项目在 memory 中的标准目录结构：

```
memory/projects/<project-name>/
├── charter.md          # 项目章程
├── plan/               # 规划文件
│   ├── scope.md        # 范围管理
│   ├── schedule.md     # 进度计划
│   ├── budget.md       # 成本预算
│   ├── risk-register.md # 风险登记册
│   └── stakeholders.md # 相关方登记
├── reports/            # 状态报告
│   ├── weekly-YYYY-MM-DD.md
│   └── status-YYYY-MM-DD.md
├── changes/            # 变更记录
│   └── change-NNN.md
├── issues/             # 问题日志
│   └── issue-NNN.md
└── closing/            # 收尾文件
    └── lessons-learned.md
```

## 日常触发场景

| 用户说 | 响应动作 |
|--------|---------|
| 「开个新项目/启动项目」 | 引导完成启动阶段，生成章程 |
| 「做项目规划/排期」 | 逐领域引导规划，生成项目管理计划 |
| 「项目进展怎么样/周报」 | 生成状态报告 |
| 「有风险/出了问题」 | 更新风险登记册或问题日志 |
| 「要变更需求」 | 启动变更控制流程 |
| 「项目要收尾了」 | 引导完成收尾阶段 |

## 方法论选择指导

根据项目特征选择管理方法：

- **预测型（瀑布）**：需求明确、技术成熟、变更少
- **适应型（敏捷）**：需求模糊、技术新颖、变更频繁
- **混合型**：部分模块需求明确（预测），部分需探索（适应）

默认使用混合型方法，除非用户明确指定。
