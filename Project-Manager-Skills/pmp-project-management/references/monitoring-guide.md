# 执行与监控阶段详细指导

## 目标

执行项目管理计划中的工作，持续监控绩效，管理变更和风险，确保项目按基准推进。

---

## 3.1 项目执行管理

### 指导与管理项目工作
- 按项目管理计划执行项目工作
- 管理技术接口、组织接口和人际接口
- 产出可交付成果、工作绩效数据、变更请求

### 管理项目知识
- 利用已有知识（组织过程资产）指导执行
- 在执行中产生新知识（经验教训、新方法）
- 使用**经验教训登记册**实时记录：什么有效、什么无效、建议

### 管理质量
- **质量审计**：独立评估质量活动和过程是否符合标准
- **过程改进**：识别过程瓶颈，持续优化
- 使用七种基本质量工具（见规划指南 2.4）分析质量问题

### 管理团队
- **塔克曼阶梯理论**：团队发展五阶段
  1. 形成期（Forming）— 礼貌但疏远，需明确规则
  2. 震荡期（Storming）— 冲突和分歧，需调解
  3. 规范期（Norming）— 建立信任和协作
  4. 执行期（Performing）— 高效运转
  5. 解散期（Adjourning）— 项目结束
- **冲突解决**：按优先级：合作/解决问题 → 妥协 → 缓和/包容 → 退避/撤退 → 强制/命令
- **激励理论**：
  - 马斯洛需求层次：生理→安全→社会→尊重→自我实现
  - 赫茨伯格双因素：保健因素（不满）+ 激励因素（满意）
  - 期望理论：动力 = 期望值 × 工具性 × 效价

### 管理沟通
- 按沟通计划分发信息
- 确保信息及时、准确、完整地传达给正确的受众
- 关注沟通效果反馈

### 管理相关方参与
- 按参与计划与相关方互动
- 处理相关方的关切和问题
- 将相关方反馈纳入项目管理考虑

## 4.1 项目监控管理

### 监控项目工作
**核心活动：** 对比实际绩效与计划基准，识别偏差

**工具与技术：**
- **挣值管理（EVM）** — 量化进度和成本偏差
  - 使用规划阶段的 EVM 指标体系
  - **完工估算（EAC）**：预测项目最终成本
    - 典型偏差（CPI 持续偏离）：EAC = BAC / CPI
    - 非典型偏差（一次性偏离）：EAC = AC + (BAC - EV)
  - **完工尚需估算（ETC）**：剩余工作还需多少成本
  - **完工偏差（VAC）**：VAC = BAC - EAC（正值=节约）
  - **完工绩效指数（TCPI）**：TCPI = (BAC-EV) / (BAC-AC)，完成剩余工作需要的成本效率

- **趋势分析**：观察 EVM 指标随时间的变化趋势
- **偏差分析**：SV、CV 的具体原因分析
- **技术绩效分析**：对比技术指标的实际值与计划值

### 实施整体变更控制（核心过程）

**变更控制流程：**

```
提出变更 → 记录变更请求 → 评估影响（范围/进度/成本/质量）
→ CCB 审批（批准/否决/推迟）→ 更新基准和文档 → 通知相关方 → 实施变更 → 验证变更
```

**关键要点：**
- 所有变更必须走正式流程，禁止私下修改
- 变更请求是书面文件，需包含：变更描述、原因、影响分析
- CCB（变更控制委员会）由关键干系人组成，负责审批
- 紧急变更可先实施后补流程，但必须在约定时间内补齐审批
- 每个变更需跟踪到关闭

**变更日志模板：**
| 变更ID | 描述 | 提出人 | 日期 | 影响分析 | 决策 | 实施状态 |
|--------|------|--------|------|---------|------|---------|
| CR-001 | | | | 范围:+X天 成本:+Y | 批准/否决 | 已实施/关闭 |

### 确认范围
- 正式验收已完成的可交付成果
- 对照范围基准和 WBS 词典中的验收标准逐项检查
- 获得客户/发起人的书面验收
- 未通过的交付物记录缺陷，走缺陷修复流程

### 控制范围
- 监控范围状态，管理范围基准变更
- **范围蔓延（Scope Creep）**：未经控制的范围扩大 → 坚决防范
- **镀金（Gold Plating）**：团队自行添加额外功能 → 明确禁止

### 控制进度
- 判断进度偏差是否需要纠正措施
- 进度压缩（赶工/快速跟进）需评估对成本和质量的影响
- 如需调整基准，走变更控制流程

### 控制成本
- 监控 EVM 指标，判断成本趋势
- 如 CPI < 1 持续恶化，需采取措施：
  1. 审查范围是否可裁剪
  2. 优化资源效率
  3. 申请追加预算（走变更流程）
  4. 调整进度计划降低峰值成本

### 控制质量
- **属性抽样 vs 变量抽样**：
  - 属性抽样：合格/不合格（如 95% 功能通过测试）
  - 变量抽样：连续数据（如响应时间在 200ms 以内）
- **控制图**：数据点超出 ±3σ 控制限 → 过程失控，需干预

### 控制风险
- 风险审计：评估风险应对措施的有效性
- 技术绩效测量：对比技术指标预测风险
- 储备分析：评估应急储备是否充足
- 风险复审会议：定期审查风险登记册

### 控制采购
- 供应商绩效监控
- 检查和审计合同执行情况
- 处理合同变更和索赔

## 问题管理 vs 风险管理

| 维度 | 风险 | 问题 |
|------|------|------|
| 时间 | 未来可能发生 | 已经发生 |
| 处理 | 应对策略（预防） | 纠正措施（补救） |
| 管理 | 风险登记册 | 问题日志 |
| 紧迫性 | 可规划 | 需立即响应 |

**问题升级路径：**
1. 团队内部解决
2. PM 协调解决
3. PM 升级到发起人
4. 发起人升级到管理层

## 监控频率建议

| 活动频率 | 活动内容 |
|---------|---------|
| 每日 | 检查问题日志，处理紧急事项 |
| 每周 | 更新进度，EVM 计算，周报 |
| 每两周 | 风险复审会议 |
| 每里程碑 | 里程碑评审，验收可交付成果 |
| 按需 | 变更评审，问题升级 |

## 关键预警信号

出现以下情况时需立即上报并启动应对：

- 🔴 关键路径上的活动延期
- 🔴 CPI 或 SPI 连续两周低于 0.9
- 🔴 关键相关方突然失去参与
- 🔴 未预见的重大风险事件发生
- 🔴 团队核心成员离职或不可用
- 🟡 应急储备使用超过 50%
- 🟡 变更请求数量异常增多
- 🟡 团队士气明显下降
