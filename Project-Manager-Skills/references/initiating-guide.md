# 启动阶段详细指导

## 目标

获得项目正式授权，明确项目边界和高层级方向。

## 核心过程

### 1.1 制定项目章程

**输入：**
- 商业论证 / 可行性分析
- 协议（合同、MOU、SOW等）
- 事业环境因素（组织文化、市场条件、法规约束）
- 组织过程资产（历史项目经验、模板、流程）

**工具与技术：**
- **专家判断**：邀请有经验的 PM、技术专家、业务专家评估
- **数据收集**：
  - 头脑风暴：快速收集干系人期望和潜在需求
  - 焦点小组：针对关键问题深入讨论
  - 访谈：一对一获取高管和关键干系人的真实想法
- **人际关系与团队技能**：
  - 冲突管理：处理干系人之间的分歧
  - 引导：确保讨论高效、聚焦
  - 会议管理：控制议程和时间
- **会议**：启动会，正式宣布项目开始

**输出：**
- 项目章程（使用 project-charter.md 模板）

**关键要点：**
- 章程由发起人签发，赋予项目经理调配资源的权力
- 目标必须符合 SMART 原则：Specific（具体）、Measurable（可衡量）、Achievable（可达成）、Relevant（相关性）、Time-bound（有时限）
- 明确高层级需求和边界：哪些在范围内，哪些明确排除

### 1.2 识别相关方

**工具与技术：**
- **相关方分析**：使用权力/利益方格分类管理

  | | 低利益 | 高利益 |
  |---|--------|--------|
  | **高权力** | 重点管理（令其满意） | 重点管理（密切参与） |
  | **低权力** | 监督（最简沟通） | 随时告知（保持满意） |

- **相关方映射**：识别内部/外部、支持者/反对者
- **会议**：与关键干系人初步沟通

**输出：**
- 相关方登记册（姓名、角色、期望、影响力评估、参与策略）

**关键要点：**
- 启动阶段的相关方识别是初步的，后续阶段需持续更新
- 特别注意识别隐性相关方（受项目间接影响的人）
- 相关方登记册是敏感文件，注意保密

## 启动阶段检查清单

- [ ] 商业论证已确认
- [ ] 项目章程已起草并获得发起人批准
- [ ] 项目经理已正式任命
- [ ] 相关方已初步识别并分析
- [ ] 项目启动会已召开，团队和关键干系人已对齐
- [ ] 章程和登记册已存档到 memory/projects/<project-name>/
